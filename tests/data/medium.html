section--first section--last"><div class="section-divider"><hr class="section-divider"></div><div class="section-content"><div class="section-inner sectionLayout--insetColumn"><h1 name="8763" id="8763" class="graf graf--h3 graf--leading graf--title">Topic Modeling with Scikit Learn</h1><p name="6de9" id="6de9" class="graf graf--p graf-after--h3">Latent Dirichlet Allocation (LDA) is a algorithms used to discover the topics that are present in a corpus. A few open source libraries exist, but if you are using Python then the main contender is <a href="https://radimrehurek.com/gensim/" data-href="https://radimrehurek.com/gensim/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Gensim</a>. Gensim is an awesome library and scales really well to large text corpuses. Gensim, however does not include Non-negative Matrix Factorization (NMF), which can also be used to find topics in text. The mathematical basis underpinning NMF is quite different from LDA. I have found it interesting to compare the results of both of the algorithms and have found that NMF sometimes produces more meaningful topics for smaller datasets. NMF has been included in Scikit Learn for quite a while but LDA has only recently (late 2015) been included. The great thing about using Scikit Learn is that it brings API consistency which makes it almost trivial to perform Topic Modeling using both LDA and NMF. Scikit Learn also includes seeding options for NMF which greatly helps with algorithm convergence and offers both online and batch variants of LDA.</p><h3 name="e1aa" id="e1aa" class="graf graf--h3 graf-after--p">How do LDA and NMF work?</h3><p name="daf0" id="daf0" class="graf graf--p graf-after--h3">I won‚Äôt go into any lengthy mathematical detail‚Ää‚Äî‚Ääthere are many blogs posts and academic journal articles that do. While LDA and NMF have differing mathematical underpinning, both algorithms are able to return the documents that belong to a topic in a corpus and the words that belong to a topic. LDA is based on probabilistic graphical modeling while NMF relies on linear algebra. Both algorithms take as input a bag of words matrix (i.e., each document represented as a row, with each columns containing the count of words in the corpus). The aim of each algorithm is then to produce 2 smaller matrices; a document to topic matrix and a word to topic matrix that when multiplied together reproduce the bag of words matrix with the lowest error.</p><figure name="909f" id="909f" class="graf graf--figure graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 700px; max-height: 393px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 56.2%;"></div><div class="progressiveMedia js-progressiveMedia graf-image" data-image-id="1*MLJVWz4EdOFsqhvBxEi9iA.png" data-width="940" data-height="528" data-action="zoom" data-action-value="1*MLJVWz4EdOFsqhvBxEi9iA.png"><img src="https://cdn-images-1.medium.com/freeze/max/38/1*MLJVWz4EdOFsqhvBxEi9iA.png?q=20" crossorigin="anonymous" class="progressiveMedia-thumbnail js-progressiveMedia-thumbnail"><canvas class="progressiveMedia-canvas js-progressiveMedia-canvas"></canvas><img class="progressiveMedia-image js-progressiveMedia-image" data-src="https://cdn-images-1.medium.com/max/1000/1*MLJVWz4EdOFsqhvBxEi9iA.png"><noscript class="js-progressiveMedia-inner"><img class="progressiveMedia-noscript js-progressiveMedia-inner" src="https://cdn-images-1.medium.com/max/1000/1*MLJVWz4EdOFsqhvBxEi9iA.png"></noscript></div></div><figcaption class="imageCaption">The resulting matrices from the NMF algorithm</figcaption></figure><blockquote name="bbcb" id="bbcb" class="graf graf--blockquote graf-after--figure">No magic here‚Ää‚Äî‚Ääyou need to specify the number of topics!</blockquote><p name="f1a0" id="f1a0" class="graf graf--p graf-after--blockquote">How many topics? Well that is the question! Both NMF and LDA are not able to automatically determine the number of topics and this must be specified.</p><h3 name="ecee" id="ecee" class="graf graf--h3 graf-after--p">Dataset Preprocessing</h3><p name="2fcc" id="2fcc" class="graf graf--p graf-after--h3">I searched far and wide for an exciting dataset and finally selected the 20 Newsgoups dataset. I‚Äôm just being sarcastic‚Ää‚Äî‚ÄäI selected a dataset that is both easy to interpret and load in Scikit Learn. The dataset is easy to interpret because the 20 Newsgroups are known and the generated topics can be compared to the known topics being discussed. Headers, footers and quotes are excluded from the dataset.</p><figure name="dfb7" id="dfb7" class="graf graf--figure graf--iframe graf-after--p"><div class="aspectRatioPlaceholder is-locked"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 35.699999999999996%;"></div><div class="iframeContainer"><IFRAME width="700" height="250" src="/media/6387e1ae228c99e95b6b483948732aa2?postId=e80d33668730" data-media-id="6387e1ae228c99e95b6b483948732aa2" allowfullscreen frameborder="0"></IFRAME></div></div></figure><p name="85b7" id="85b7" class="graf graf--p graf-after--figure">The creation of the bag of words matrix is very easy in Scikit Learn‚Ää‚Äî‚Ääall the heavy lifting is done by the feature extraction functionality provided for text datasets.
